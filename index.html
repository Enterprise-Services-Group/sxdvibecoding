Step 1

Getting Started & Setup

Get Copilot running in VS Code in under a minute.

1

Sign In

Hover over the Copilot icon in the Status Bar and select Use AI Features. Sign in with your GitHub account — your Enterprise subscription activates automatically.

2

Open Chat

Press ⌃⌘I (macOS) or Ctrl+Alt+I (Win/Linux) to open the Chat view. This is your main AI conversation hub.

3

Initialize Your Project

Type /init in chat. Copilot analyzes your codebase and generates a copilot-instructions.md file matched to your project's patterns.

🧪 Try It

Open any project in VS Code, press ⌃⌘I, type /init, and review the generated instructions. Commit the file to share conventions with your team.

Step 2

Inline Suggestions (Ghost Text)

Copilot offers dimmed ghost-text completions as you type — from single lines to entire function bodies.

How It Works

Start typing in any language and Copilot generates context-aware suggestions drawing from your open files, project structure, and coding patterns.

  • Tab — Accept the full suggestion
  • ⌘→ — Accept the next word only
  • ⌥] / ⌥[ — Cycle through alternative suggestions
  • Esc — Dismiss the suggestion
  • Hover over ghost text to see navigation controls
function calculateDaysBetween(start, end) { const diffTime = Math.abs(end - start); const diffDays = Math.ceil(diffTime / (1000 * 60 * 60 * 24)); return diffDays; }
💡

Pro tip: Write a descriptive code comment (e.g. // use recursion to flatten nested arrays) and Copilot generates an implementation matching your intent. Open related files in your editor to give Copilot better context.

⚙️ Settings & Configuration

  • github.copilot.enable — Enable/disable per language ({ "python": true, "markdown": false })
  • editor.inlineSuggest.fontFamily — Change the ghost text font
  • editor.inlineSuggest.syntaxHighlightingEnabled — Toggle syntax highlighting for ghost text
  • Use the Copilot menu in the Status Bar to snooze suggestions temporarily (5-minute increments)
  • Run GitHub Copilot: Change Completions Model from the Command Palette to switch the AI model used for inline suggestions
🧪 Try It

Create a new file, write // A function that generates a random hex color on the first line and press Enter. Watch Copilot generate the implementation. Press ⌥] to see alternative versions.

Step 3

Next Edit Suggestions (NES)

Powerful

NES predicts where your next edit should be and what it should change — then lets you Tab through edits sequentially.

🔤

Fix Typos & Mistakes

Copilot catches typos like conts → const, inverted ternary logic, or || that should be &&.

🎯

Follow Intent Changes

Rename Point to Point3D, and NES suggests adding a z property, then updating the distance calculation.

♻️

Propagate Refactors

Rename a variable once and NES suggests updating every subsequent reference. Paste code, and it adjusts to match the local style.

➡️

Navigation: An arrow in the gutter shows where the next edit suggestion is. Press Tab to jump to the suggestion, then Tab again to accept it. Hover the gutter arrow for the full menu.

⚙️ NES Settings

  • github.copilot.nextEditSuggestions.enabled — Enable NES (must be on)
  • editor.inlineSuggest.edits.allowCodeShifting — Allow NES to shift code to display a suggestion
  • editor.inlineSuggest.edits.renderSideBySide — Show larger edits side-by-side (auto or never)
  • github.copilot.nextEditSuggestions.fixes — Enable NES based on diagnostics (squiggles, missing imports)
  • editor.inlineSuggest.edits.showCollapsed — Show changes only when you Tab to them (reduce visual noise)
🧪 Try It

Open a file with a class. Rename one property, then watch for the gutter arrow suggesting the next place that needs updating. Tab through the chain of related edits.

Step 4

Chat View

Your main AI conversation hub — multi-turn conversations, file edits, terminal commands, and more.

Opening Chat

  • ⌃⌘I — Open the Chat view in the sidebar
  • ⌘N — Start a new chat session
  • ⌥⌘. — Open the model picker to switch AI model

Session Configuration

Each session has four key settings:

  • Session type — Local, Background, or Cloud
  • Agent — Agent, Plan, Ask, or custom agents
  • Permission level — Default Approvals, Bypass Approvals, or Autopilot
  • Language model — Choose from available models via the model picker
Prompt (Agent)

Create a REST API with Express.js
that has endpoints for users and
products with input validation.

→ Creating server.js...
→ Creating routes/users.js...
→ Creating routes/products.js...
→ Running npm install...
→ Running tests... ✓ All passed

Adding Context to Prompts

#

#-mentions

Type # to reference files, folders, symbols, tools, or variables like #codebase, #changes, #problems, #terminalSelection.

@

@-mentions

Type @ for domain-specific participants: @github (repos, PRs, issues), @terminal (shell commands), @vscode (editor features).

📎

Drag & Drop

Drag files, folders, editor tabs, or Problems panel items directly into chat. Attach images/screenshots for visual context.

Reviewing Changes

📝

Inline Diffs

Open a changed file to see inline diffs. Use the editor overlay to navigate between edits and Keep or Undo individually.

📸

Checkpoints

VS Code auto-creates snapshots at key points. Roll back to any previous state if the agent goes off track.

📂

Stage to Accept

Staging changes in Source Control automatically accepts pending edits. Discarding changes discards edits too.

💡

Multi-turn tip: Build on previous responses within the same session. The AI maintains full context. Use /compact to summarize when the conversation grows long, or /fork to branch off into a new independent session.

🧪 Try It

Open Chat, select the Agent mode, and prompt: "Create a calculator web app with HTML, CSS, and JS. Add unit tests." Watch the agent create files, install dependencies, and run tests autonomously.

Step 5

Inline Chat & Quick Chat

Targeted AI assistance without leaving the editor or terminal. Perfect for quick, focused changes.

📝

Editor Inline Chat

Press ⌘I in the editor to open a prompt directly in your code. Select a code block first to scope the prompt. Changes appear as inline diffs — Keep or Undo.

⬛

Terminal Inline Chat

Press ⌘I in the integrated terminal. Ask for shell commands in natural language. Review the suggestion, then ⌘Enter to run or ⌥Enter to insert for editing.

⚡

Quick Chat

Press ⇧⌥⌘L for a lightweight popup chat at the top of the editor. Ask quick questions without opening the full Chat view. Continue in full view if needed.

💡

When to use what: Use inline chat for targeted single-file edits (refactoring a function, adding error handling). Use the Chat view for multi-file tasks. Use Quick Chat for one-off questions.

🧪 Try It

Select a function in your code, press ⌘I, and type "Add input validation and error handling". Review the inline diff and keep the changes. Then open the terminal, press ⌘I, and ask "find all TODO comments in this project".

Step 6

Agents — Autonomous AI Assistants

Give an agent a goal and it plans, edits files, runs commands, and self-corrects across your entire project.

Built-in Agents

🤖

Agent

Autonomously plans and implements changes across multiple files. Runs terminal commands, invokes tools, and iterates on failures. The default workhorse.

📋

Plan

Creates a structured, step-by-step implementation plan before any code is written. Review the plan, refine it, then hand off to Agent for execution.

❓

Ask

Answers questions about your codebase, coding concepts, or VS Code itself — without making file changes. Read-only exploration and learning.

Agent Types — Where They Run

💻

Local Agents

Run interactively in your VS Code editor with full access to workspace, tools, extensions, and MCP servers. Best for iterative, hands-on work.

⬛

Copilot CLI (Background)

Run autonomously on your machine from the command line. Optionally use Git worktrees for isolation. Great for well-defined tasks you don't need to watch.

☁️

Cloud Agents

Run remotely on GitHub. Create branches, implement changes, and open pull requests for team review. Assign GitHub issues directly to copilot.

🔌

Third-Party Agents

Use agents from Anthropic (Claude) or OpenAI (Codex) through the VS Code agent harness. Run locally or in the cloud with your Copilot subscription.

Permission Levels

🔒

Default Approvals

Only read-only/safe tools are auto-approved. You confirm each write operation and terminal command.

⚡

Bypass Approvals

Auto-approves all tool calls without confirmation dialogs. The agent may still ask clarifying questions.

🚀

Autopilot Preview

Auto-approves everything and auto-responds to questions. The agent works fully autonomously until the task is done.

🔄

Hand-off: Start with Plan to design an approach, hand off to a local Agent for implementation, then delegate to a Cloud agent to open a PR. Conversation history carries over at each step. Run multiple sessions in parallel via the Sessions list.

🧪 Try It

Select Plan from the agent picker and prompt: "Plan a migration of our test suite from Jest to Vitest." Review the plan, then change the agent to Agent and let it execute. Use checkpoints to roll back if needed.

Step 7

Smart Actions

AI-powered actions integrated directly into the VS Code UI — no prompt required. They appear contextually where you need them.

💬

Generate Commit Messages

Click the sparkle icon in Source Control to generate a commit message from your staged changes. Also generates PR titles and descriptions.

🔀

Resolve Merge Conflicts Experimental

Click Resolve Merge Conflict with AI in the editor. Copilot uses the merge base and both branch changes to propose a resolution.

📖

Generate Documentation

Right-click → Generate Code → Generate Docs. Copilot writes documentation comments for your selected code in the appropriate format.

🧪

Generate Tests

Right-click → Generate Code → Generate Tests. Tests are added to an existing test file or a new one is created automatically.

🔧

Fix Coding Errors

Right-click → Generate Code → Fix. Also: watch for the lightbulb Code Action on compiler/linting errors with AI fix suggestions.

❌

Fix Test Failures

In the Test Explorer, hover a failing test and click the sparkle icon. Or use /fixTestFailure in chat. Agents auto-fix and re-run when enabled.

⬛

Fix Terminal Errors

When a command fails, a sparkle appears in the terminal gutter offering a Quick Fix to explain what happened and suggest a corrected command.

🔍

Code Review

Right-click → Generate Code → Review for a code block, or use the Code Review button in the PR Files Changed view for all changes.

✏️

Rename with AI

Press F2 to rename a symbol and Copilot suggests contextually appropriate names based on usage and surrounding code.

🔎

Semantic Search Preview

Find results in the Search view that are semantically relevant even without exact text matches. Enable with search.searchView.semanticSearchBehavior.

🖼️

Alt Text for Images

In Markdown files, put cursor on an image link → Code Action lightbulb → Generate alt text. Also supports Refine alt text.

✅

Implement TODOs

Add a TODO comment in code. Click the Code Action → Delegate to coding agent. Copilot coding agent creates a PR implementing the task.

🧪 Try It

Stage some changes in Source Control and click the sparkle icon to generate a commit message. Then select a function, right-click → Generate Code → Generate Tests to auto-generate a test file.

Context & Built-in Tools

Providing the right context is the single most important thing for getting quality AI responses. Here are all the context mechanisms available.

# Context Variables (type # in chat)

#fileReference a specific file
#folderReference a folder and its contents
#symbolReference a code symbol (function, class, etc.)
#codebasePerform automatic code search across workspace
#changesCurrent source control changes
#problemsWorkspace errors from the Problems panel
#selectionCurrent editor text selection
#terminalSelectionCurrent terminal selection
#searchResultsResults from the Search view
#fetchFetch content from a web URL
#extensionsSearch/ask about VS Code extensions

Built-in Tool Sets (used by agents)

#readRead files, notebook summaries, terminal output
#editCreate/edit files and directories, edit notebooks
#executeRun terminal commands, notebook cells, tasks
#searchText search, file search, codebase search, list directories
#agentDelegate tasks to other agents / run subagents
#browserNavigate, screenshot, click, type in integrated browser Experimental
#webFetch web page content (#web/fetch)
#vscodeRun commands, install extensions, search extensions, VS Code API
#todosTrack implementation progress with a todo list
💡

Tool management: Click the Configure Tools button in chat to see all available tools and toggle them on/off. Fewer active tools means faster, more relevant responses.

Step 11

Customization & Configuration

Tailor the AI to your project so it generates code matching your conventions from the start.

📜 Custom Instructions

Define coding standards that automatically apply to every AI response.

Always-on Instructions (apply to every request)

  • .github/copilot-instructions.md — Project-wide coding conventions. Start here.
  • AGENTS.md — Cross-agent instructions (works with Claude, Copilot, etc.)
  • CLAUDE.md — Claude-specific compatibility (also recognized by VS Code)
  • Organization-level — Shared across all repos in your GitHub org

File-based Instructions (conditional, with applyTo patterns)

--- name: 'Python Standards' description: 'Coding conventions for Python files' applyTo: '**/*.py' --- # Python coding standards - Follow PEP 8 style guide - Use type hints for all function signatures - Write docstrings for public functions

Store .instructions.md files in .github/instructions/. Organize by language, team, or module. VS Code searches recursively.

Quick Commands

  • /init — Auto-generate instructions from your project
  • /instructions — Open the Configure Instructions menu
  • /create-instruction — Generate a targeted instructions file with AI

🤖 Custom Agents

Create specialized AI personas with their own tools, instructions, and model preferences.

--- name: 'Reviewer' description: 'Review code for quality and best practices.' tools: ['read', 'search', 'web'] --- # Code Reviewer Agent You are an experienced senior developer. Review code for quality, security issues, and performance problems. ## Important - DO NOT make direct code changes - Focus on explaining what and why - Ask clarifying questions about design decisions

Save as .github/agents/Reviewer.agent.md. Appears in the agent picker. Use /create-agent to generate one with AI.

🔌 MCP Servers (Model Context Protocol)

Extend agents with external tools — databases, APIs, browsers, and more via the open MCP standard.

Install from Gallery

  • Open Extensions view (⇧⌘X) → search @mcp
  • Click Install to add to your user profile, or right-click → Install in Workspace

Configure Manually

// .vscode/mcp.json { "servers": { "github": { "type": "http", "url": "https://api.githubcopilot.com/mcp" }, "playwright": { "command": "npx", "args": ["-y", "@microsoft/mcp-server-playwright"] } } }

Capabilities

  • Tools — Actions like file ops, DB queries, API calls
  • Resources — Read-only context (files, tables, API responses)
  • Prompts — Preconfigured prompt templates from the server
  • MCP Apps — Interactive UI components (forms, visualizations) rendered in chat
  • Sandboxing — On macOS/Linux, restrict file system and network access per server

📁 Prompt Files, Skills & Hooks

  • Prompt Files — Reusable prompt templates you can run with /<prompt-name>. Define what task to perform (vs instructions which say how). Use /create-prompt to generate.
  • Skills — Domain-specific capabilities that work across VS Code, CLI, and cloud agents. Define specialized knowledge and tool usage patterns. Use /create-skill to generate.
  • Hooks — Execute custom commands at lifecycle events (before/after tool calls, on session start/end). Use for automation and policy enforcement. Use /create-hook to generate.

Manage all customizations from the Chat Customizations editor: run Chat: Open Chat Customizations from the Command Palette.

🧪 Try It

Type /init in chat to generate instructions for your project. Then type /create-agent code reviewer that focuses on security to create a custom agent. Try switching to it in the agent picker.

Best Practices & Prompt Engineering

Proven strategies for getting the best results from AI in VS Code.

Pick the Right Tool for the Task

Tool Best For Examples

Ask (chat)Questions, brainstorming"How does auth work in this project?"
Inline ChatTargeted, in-place editsRefactoring a function, adding error handling

PlanStructured planningArchitecture design, migration strategy
Smart ActionsOne-step specialized tasksCommit messages, rename, fix errors

Writing Effective Prompts

🎯

Be Specific

State the language, frameworks, expected behavior, and constraints. Include example input/output. Vague prompts like "make this better" give the AI no direction.

🧩

Break Down Tasks

Decompose complex features into smaller steps. This produces more reliable results and catches problems early. Let multi-turn conversations build on each step.

✅

Include Verification

Provide test cases and acceptance criteria. "Write unit tests that verify: 10 req/s allowed, 11th rejected, bucket refills after 1s." This is the highest-leverage thing you can do.

🔄

Iterate & Course-Correct

Refine with follow-up messages. If the AI heads wrong, steer it with a new prompt rather than starting over. Tell it to ask clarifying questions if the task is ambiguous.

The Plan → Implement → Review Workflow

1

Explore

Use Ask mode or subagents to read code and understand how it works before making changes.

2

Plan

Use the Plan agent to create a structured implementation plan. Review and refine before executing.

3

Implement

Switch to Agent and implement from the plan. Include tests. Hand off to background or cloud for long tasks.

4

Review

Use checkpoints to review progress. Rewind if off track. Request a Copilot code review on the PR.

💡

Session hygiene: Start new sessions for unrelated tasks — context pollution degrades responses. Use /compact to trim long conversations. Run parallel sessions for independent work. Use subagents to explore in isolation without cluttering your main context.

🧪 Try It

Switch to Plan agent and prompt: "Plan adding dark mode support to this app. Consider CSS variables, user preference persistence, and system theme detection." Review the plan, then hand off to Agent.
